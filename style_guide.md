# Художественные требования
Для удобства ссылок положения данного соглашения будут пронумерованы.

## I. Двухмерные графические элементы
Включают в себя: текстуры блоков и моделей, спрайты предметов, графические элементы GUI и
иные другие изображения.
1. **Общие положения для текстур**:
	1. Основным разрешением текстур является соотношение 16px на 16px;
	2.В порядке исключения допустимо использовать дополнительное разрешение текстур для блоков и предметов;
	3. Дополнительным разрешением текстур является соотношение 32px на 32px;
	4. В порядке исключения допустимо использовать нечетные и нестандартные разрешения текстур для частиц и GUI;
	5. Все текстуры перед добавлением в игру должны пройти индексацию цвета через утилиту optipng;
	6. Рекомендуется: использовать ограничение палитры текстур определенным набором позиций. Рекомендательный набор для одного оттенка: от 3 до 5;
	7.Палитра текстур должна использовать сдвиг по тону (hue shifting);
2. **Положения для текстур предметов (в инвентаре)**:
	1. Текстуры предметов должны обладать легко различимым образом, не допускать двойственности интерпретации;
	2. Текстуры предметов должны обладать рамкой (outline);
	3. Рамка на текстуре предмета должна подчиняться тону самой текстуры и обладать свето-теневым переходом;
3. **Положения для текстур блоков**:
	1. Блоки, не предполагающие четко различимого паттерна, должны обладать бесшовностью;
4. **Положения для текстур моделей**:
	1. Разрешение текстур (разверток) моделей может быть не равно разрешениям, установленным в пункте “1. Общие положения для текстур”;
	2. Разрешение текстур (разверток) модели рекомендуется создавать равным значениям степеней двойки (8, 16, 32, 64, 128 и т.д.);

## II. Трехмерные графические элементы
Включают в себя: трехмерные модели блоков и сущностей.
1. **Общие положения для моделей**:
	1. В случаях, когда модель является примитивной, рекомендуется в целях повышения производительности использовать nodebox;
	2. Используемыми форматами моделей являются: *.obj или .glb* (под вопросом);
	3. Исходные файлы моделей (проекты Blender, Blockbench и др.) должны быть сохранены в репозитории;
	4. *В моделях запрещено использовать скругленные или шарообразные элементы, фаски, формы многогранников. Модель должна строиться из плоскостей и/или параллелепипедов;*
	5. Грани и плоскости модели должны быть строго кратны пикселям их текстур;
	6. Масштаб развертки модели должен быть един по всей поверхности;
	7. Масштаб развертки модели рекомендовано должен соотноситься с основным разрешением текстур как 1 к 1 или быть визуально близким к этому значению.
